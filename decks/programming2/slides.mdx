## Authentication

* No tokens, no crypto
  * Client library authentication **not enough**
* How?
  * Trusted endpoints add CA name to request
  * Injected requests cannot be modified
* Implementation
  * Kubernetes network policy
  * Security proxies in plugins

---

<CafjsHeader/>

## Authorization

* Access control check at the destination
  * **before** calling a CA method
* Make the caller CA name visible while the method executes
* Also enforced for external calls (client library)

---

<CafjsHeader/>

## Default policy

### <ins>Trust No One</ins>

* Only the owner can invoke methods using the client library
  * No *Trusted Bus* interactions
* If nothing works...
  * check the security policy first!

## SharedMap

---

<CafjsHeader/>

### CA + Distributed Data Structure (DDS)

* Efficient data replication that respects CA serialization
* CA (Actor Model)
  * Keep private copies updated with requests &#x2639;
* DDS
  * Use shared-memory for replicated data &#x263A;
* CA + DDS
  * DDS visible as private CA state
  * Avoid shared-memory <ins>GREMLINS</ins>
    * races, deadlocks, awful failure modes...
  * Observational equivalence
    * Can replace DDS by extra requests

---

<div>

**SharedMap**

* Single writer/multiple reader
* Name relative to CA
  * antonio-ca1-map1
* Silent updates
* Not an ES6 Map
* Browser and IoT

```js
exports.methods = {
    async __ca_init__() {
        if (isAdmin(this)) {
            const initV = {counter: 0};
            this.$.sharing.addWritableMap('primary', ADMIN_MAP, {initV});
        }
        this.$.sharing.addReadOnlyMap('replica', primaryMap(this));
        return [];
    },
    async __ca_pulse__() {
        if (isAdmin(this)) {
            const $$ = this.$.sharing.$;
            const counter = $$.primary.get('counter');
            $$.primary.set('counter', counter + 1);
        }
        return [];
    },
    async getCounter() {
        const value = this.$.sharing.$.replica.get('counter');
        return [null, value];
    }
};
```

### Updating code

* A *SharedMap* can contain serialized methods
  * A custom replicated object!
* Taming `eval`
  * `setFun()`, `applyMethod()`, and `{noExec: true}`
  * Single writer + naming convention + `Trusted Bus`
    * Authenticates *SharedMap* custom method origin
* Transactional updates of data and code
* Many uses:
  * getters/setters to hide schema changes
  * Dynamic code updates for devices
  * Choose where code runs

<div>

**Publish-subscribe**

```js
exports.methods = {
    async __ca_init__() {
        this.state.counter = 0;
        this.$.pubsub.subscribe(mainChannel(this), '__ca_handle__');
        this.$.security.addRule(this.$.security.newSimpleRule(
            '__ca_handle__', this.$.security.SELF, ADMIN_CA
        ));
        return [];
    },
    async __ca_pulse__() {
        if (isAdmin(this)) {
            this.state.counter = this.state.counter + 1;
            this.$.pubsub.publish(mainChannel(this),
                                  `Counter: ${this.state.counter}`);
        }
        return [];
    },
    async __ca_handle__(topic, msg, from) {
        this.$.session.notify([msg]);
        return [];
    }
};
```
